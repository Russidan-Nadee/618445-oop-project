# System API Overview

ระบบนี้แบ่ง API ออกตาม module ของข้อมูล  
แต่ละ module จะมี API สำหรับจัดการข้อมูลของตัวเองโดยตรง  
รายละเอียดของแต่ละ module จะถูกแยกไปเขียนเป็นไฟล์เอกสารเฉพาะ

ปัจจุบันระบบมี API หลักทั้งหมด **5 ชุด**

---

## API modules

### 1. Assets API  
**Doc:** `assets.api.md`

API สำหรับจัดการข้อมูลทรัพย์สิน เช่น การเพิ่ม แก้ไข ลบ และดึงข้อมูล  
ดูแลข้อมูลของ Asset และความสัมพันธ์กับ Asset Type  
การจัดการ Asset โดยตรงไม่สร้าง log

---

### 2. Asset Types API  
**Doc:** `asset-types.api.md`

API สำหรับจัดการประเภทของทรัพย์สิน  
ใช้สำหรับจัดหมวดหมู่ Asset และเป็นข้อมูลอ้างอิงในการสร้างทรัพย์สิน

---

### 3. Users API  
**Doc:** `users.api.md`

API สำหรับจัดการข้อมูลผู้ใช้งานในระบบ  
User ถูกใช้เป็นข้อมูลอ้างอิงใน Transaction และ Log

---

### 4. Transactions API  
**Doc:** `transactions.api.md`

API สำหรับการยืมและคืนทรัพย์สิน  
เป็นจุดที่เกิดการใช้งานทรัพย์สินจริงในระบบ

เมื่อมีการยืมหรือคืน:
- สถานะของ Asset จะถูกอัปเดต
- ระบบจะบันทึก log ของการกระทำนั้น

---

### 5. Logs API  
**Doc:** `logs.api.md`

API สำหรับดูประวัติการยืม–คืนที่เกิดขึ้นในระบบ  
log จะถูกสร้างเฉพาะจากการยืมและคืนเท่านั้น

---

## Notes

รายละเอียดของ endpoint แต่ละตัว  
จะถูกอธิบายไว้ในไฟล์เอกสารของแต่ละ module
